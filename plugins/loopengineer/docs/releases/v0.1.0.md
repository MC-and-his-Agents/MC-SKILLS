# v0.1.0 Manual Release Plan

Issue: #47

`v0.1.0` is the first manual release checkpoint for LoopEngineer. It records the
context safety minimum version, plugin skeleton metadata, and lightweight
routing baseline. It does not mean the full orchestration protocol is stable.

## Included Scope

- Context safety budget schema and defaults.
- Context guard script and tests.
- No-inline large artifact policy.
- Handoff manifest and rotation guidance.
- Plugin skeleton metadata.
- Lightweight loop router.
- Heavy trigger policy baseline.

## Not Included

- Stable scheduler, watcher, audit, or release runtime.
- MCP servers, hooks, automations, or Loom adapter install/register behavior.
- Package-manager publishing.
- GitHub Release artifacts.
- Automatic tagging.

## Checklist

- `VERSION` contains `0.1.0`.
- `metadata/loopengineer.json.version` matches `VERSION`.
- `.codex-plugin/plugin.json.version` matches `VERSION`.
- `CHANGELOG.md` has a `0.1.0` entry with compatibility fields.
- Existing `skills/*/skill.yaml` files declare `version` and
  `skillContractVersion`.
- Existing `schemas/v*/...schema.json` files declare `$id`, `schemaVersion`,
  and `kind`.
- Skill entrypoints under `skills/` are present and reference existing files.
- Schema examples under `schemas/v1/examples/` remain valid or intentionally
  invalid according to tests.
- Tests pass with `python3 -m unittest discover -s tests`.
- Diff hygiene passes with `git diff --check`.

## Manual Tag Rule

After the checklist passes on the release commit, create the tag manually:

```text
git tag v0.1.0
git push origin v0.1.0
```

Do not create this tag as part of #47. The tag command is documentation for the
future manual release action.

## Release Notes Shape

Use this structure for release notes:

```markdown
## Summary
## Scope
## Validation
## Compatibility
## Known Limitations
```

The compatibility section must include:

- pluginApiVersion
- protocolVersion
- schemaMajorVersion
- skillContractVersion
- adapterContractVersion

## Pre-Release Checks

Run from the repository root:

```text
git status --short --branch
git diff --check
python3 -m unittest discover -s tests
```

If any check fails, classify the failure before retrying: code/documentation
contract, schema/example mismatch, environment/tooling, or permissions.
