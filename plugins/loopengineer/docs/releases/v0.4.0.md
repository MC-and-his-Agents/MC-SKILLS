# v0.4.0 Manual Release Plan

Issue: #36

`v0.4.0` is the M5 release checkpoint for LoopEngineer. It records loop audit,
coordination cost, watcher inbox, and watcher policy capabilities. It does not
mean a stable autonomous watcher runtime exists.

## Included Scope

- Lazy shared channel policy.
- Watcher inbox schema, examples, validation, digest support, and design doc.
- Watcher automation mode and rotation policy.
- Coordination cost design and `scripts/coordination_tax.py`.
- Loop audit skill and `scripts/loop_audit.py`.
- Version, metadata, plugin manifest, and changelog update to `0.4.0`.

## Not Included

- MCP servers, hooks, or watcher automation creation.
- Scheduler or watcher thread creation.
- Loom adapter install/register behavior.
- Package-manager publishing.
- Any release beyond the Git tag and GitHub Release.

## Checklist

- `VERSION` contains `0.4.0`.
- `metadata/loopengineer.json.version` matches `VERSION`.
- `.codex-plugin/plugin.json.version` matches `VERSION`.
- `CHANGELOG.md` has a `0.4.0` entry with compatibility fields.
- Existing `skills/*/skill.yaml` files declare `version` and
  `skillContractVersion`.
- Existing `schemas/v*/...schema.json` files declare `$id`, `schemaVersion`,
  and `kind`.
- Skill entrypoints under `skills/` are present and reference existing files.
- Schema examples under `schemas/v1/examples/` remain valid or intentionally
  invalid according to tests.
- Tests pass with `python3 -m unittest discover -s tests`.
- Diff hygiene passes with `git diff --check`.
- Release readiness passes with `python3 scripts/check_release_readiness.py`.

## Manual Release Rule

After the checklist passes on `main`, dispatch `.github/workflows/manual-release.yml`
with:

```text
release_version: v0.4.0
allow_explicit_commit: false
draft_release: false
```

The workflow creates the tag and non-draft GitHub Release only after release
readiness passes.

## Release Notes Shape

Use this structure for release notes:

```markdown
## Summary
## Scope
## Validation
## Compatibility
## Known Limitations
```

The compatibility section must include:

- pluginApiVersion
- protocolVersion
- engineContractVersion
- schemaMajorVersion
- skillContractVersion
- adapterContractVersion
