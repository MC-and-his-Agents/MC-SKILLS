# v0.6.0 Manual Release Plan

Issue: #95

`v0.6.0` is the M7 release checkpoint for LoopEngineer. It records
subagent-backed `worker_lite` provider support while preserving the control
plane boundary: subagent is a bounded execution provider, not a runtime,
scheduler, watcher, gate, merge, release, or closeout owner.

## Included Scope

- ADR 0005 for the 2026-06-17 subagent/thread experiment.
- `worker_lite` provider matrix: `direct`, `subagent`, and `thread`.
- Subagent assignment schema, examples, and structure validation.
- Subagent report provider context and eligibility checks.
- `provider_selection.py` and `loopengineer.py provider-select`.
- Optional assignment-aware `consume_report.py` path for subagent receipts.
- Version, metadata, plugin manifest, changelog, and roadmap update to `0.6.0`.

## Not Included

- Real subagent spawning.
- Watcher runtime.
- Scheduler runtime.
- MCP mutation tools.
- Worker, scheduler, watcher thread creation.
- Guardian, merge-ready, merge, release, or closeout automation.
- Loom `.loom` writes, Loom CLI replacement, or Loom adapter install/register.
- GitHub Release execution.

## Compatibility

- pluginApiVersion: 1
- protocolVersion: 1
- engineContractVersion: 1
- schemaMajorVersion: 1
- skillContractVersion: 1
- adapterContractVersion: 0

`engineContractVersion: 1` remains compatible because M7 adds a read-only
diagnostic provider recommendation command and does not remove existing engine
commands. `adapterContractVersion: 0` still means there is no stable external
runtime adapter contract.

## Manual Release Rule

After the checklist passes on `main`, dispatch
`.github/workflows/manual-release.yml` with:

```text
release_version: v0.6.0
allow_explicit_commit: false
draft_release: false
```

The workflow creates the tag and GitHub Release only after release readiness
passes. Package publication remains out of scope.
